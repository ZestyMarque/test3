# Перечень протестированных пользовательских сценариев и описание их значимости.

Для выполнения данной работы был взят проект из предыдущего задания: небольшое веб-приложение на Python с использованием FastAPI и SQLite, позволяющее создавать, просматривать, завершать и удалять задачи. 
В ходе лабораторной работы были протестированы ключевые пользовательские сценарии веб-приложения для работы с задачами. 
Были выбраны три основных сценария: создание новой задачи, завершение задачи и удаление задачи. Сценарий создания задачи проверяет возможность пользователя добавлять новые задачи и корректное сохранение данных в базе. 
Сценарий завершения задачи проверяет, что статус задачи меняется правильно и эта информация отражается в системе. 
Сценарий удаления задачи позволяет убедиться, что пользователь может удалять задачи, а система корректно реагирует на удаление и обновляет состояние данных.
Эти сценарии критичны, так как без их корректной работы приложение не будет выполнять основные функции для пользователя.

# Обоснование выбора этих сценариев.

Выбор этих сценариев был обусловлен их значимостью для пользовательского опыта. 
Проверка создания, изменения и удаления задач охватывает полный путь данных от пользовательского запроса через API до записи и обновления в базе данных. 
Такой подход позволяет убедиться, что система работает целиком, а не только отдельные модули. 
Кроме того, тестирование всех этих сценариев помогает выявить возможные проблемы с логикой приложения, некорректным поведением API или ошибками взаимодействия с базой данных.

# Примеры кода тестов.

Примеры кода тестов приведены ниже. 
Тесты написаны на Python с использованием библиотеки requests и имитируют действия пользователя через HTTP-запросы к серверу:

```
def test_user_creates_task():
    response = requests.post(
        f"{BASE_URL}/tasks/",
        json={"title": "E2E Task", "description": "task from end to end test"}
    )
    assert response.status_code == 200
    data = response.json()
    assert data["title"] == "E2E Task"
    assert data["completed"] == False

def test_user_completes_task():
    tasks_resp = requests.get(f"{BASE_URL}/tasks/")
    tasks = tasks_resp.json()
    task_id = tasks[0]["id"]
    complete_resp = requests.put(f"{BASE_URL}/tasks/{task_id}")
    assert complete_resp.status_code == 200
    completed_task = complete_resp.json()
    assert completed_task["completed"] == True

def test_user_deletes_task():
    tasks_resp = requests.get(f"{BASE_URL}/tasks/")
    tasks = tasks_resp.json()
    task_id = tasks[0]["id"]
    delete_resp = requests.delete(f"{BASE_URL}/tasks/{task_id}")
    assert delete_resp.status_code == 200
    result = delete_resp.json()
    assert result["message"] == "Task deleted"
```

Результаты выполнения тестов показали, что все сценарии были успешно пройдены. 
Сервер корректно обрабатывал создание, завершение и удаление задач, статусы задач обновлялись правильно, а данные корректно сохранялись и удалялись из базы. 
В ходе тестирования ошибок и сбоев обнаружено не было, все три теста завершились успешно, что подтверждает стабильную работу системы.

Таким образом, можно сделать вывод, что пользовательский опыт в приложении удовлетворительный, ключевые функции работают корректно, а интерфейс API реагирует на запросы предсказуемо. 
Потенциальные проблемы могут возникнуть при масштабировании приложения или при одновременном доступе нескольких пользователей, но для текущей тестовой среды и учебного проекта система ведет себя стабильно и безопасно для использования.
